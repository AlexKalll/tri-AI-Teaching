Static vs Contextual Embeddings

How do we represent words as numbers? From Word2Vec's fixed vectors to BERT's context-aware representations — and why it matters for African NLP.

● TRI AI / Google DeepMind ● Session 2

Two Eras of Word Representation

⚖

Static Embeddings

Word2Vec · GloVe · FastText
Core Idea
  • One vector per word — same for all meanings
  • Trained once on a large corpus, then frozen
  • Words with similar meanings have similar vectors
  • Semantic relationships are captured as vector arithmetic
Good For

Simple similarity Word analogies Feature extraction Baseline models

Classic Example
vector("king") - vector("man") + vector("woman")
≈ vector("queen")
Limitations

Polysemy (the "bank" problem) No context awareness Fixed vocabulary (OOV)

The "Bank" Problem
"I went to the bank to withdraw money."
"The river bank was muddy."

Both get the SAME vector in static embeddings.
⚙

Contextual Embeddings

BERT · GPT · Gemma · T5
Core Idea
  • Different vector per word per context
  • Pre-trained on massive data, can be fine-tuned
  • Uses attention to look at surrounding words
  • Subword tokenization handles unseen words
Good For

Understanding nuance Machine translation Question answering Sentiment analysis

Context Matters
"I went to the bank to withdraw money."
→ vectorfinancial

"The river bank was muddy."
→ vectorgeographic

Different contexts = different vectors!
How It Works
  • Input: sequence of subword tokens
  • Transformer layers apply self-attention
  • Each token's representation is contextualized by all others
  • Output: one vector per token, informed by context

🌎 The Map of Meaning Metaphor

Imagine words as locations on a map. Static embeddings are like a paper atlas: every word has a fixed coordinate. "Bank" appears once on the map, even though it means two different things. You cannot look up different meanings for different sentences.

Contextual embeddings are like Google Maps with live traffic and context layers. The same address can show different information depending on the time of day, your destination, and current conditions. "Bank" adjusts its coordinates based on the words around it.

This matters for African languages where a single word can carry many meanings depending on tone, context, and regional dialect. Static embeddings collapse this richness into one point. Contextual embeddings preserve it.

🛠 How Contextual Embeddings Are Built

Token IDs → Token Embeddings → Position Encoding → Transformer Blocks
(Self-Attention + FFN) → Contextual Vectors
(one per token)

The key innovation: each token's final vector depends on ALL tokens in the sequence via self-attention

📄 Comparison Table

Dimension Static (Word2Vec) Contextual (BERT/Gemma)
Vectors per word One (fixed) One per context (dynamic)
Handles polysemy No — "bank" has one meaning Yes — meaning changes with context
Training Trained once, frozen Pre-trained + fine-tuned
Out-of-vocabulary words Unknown token or fail Subword tokenization handles them
Compute needed Low (shallow model) High (deep Transformer)
Data needed Moderate (~10B tokens) Massive (~100B+ tokens)
Transfer learning As input features only Full fine-tuning or adapters
Interpretability Direct word relationships Complex attention patterns

🇧🇪 African Context Examples

📖 Yoruba Legal Translation

The Yoruba word "igba" can mean time, two hundred, or garden egg depending on tone and context. Static embeddings lose this distinction. Contextual embeddings preserve it, enabling accurate legal document translation.

🌾 Agricultural Advisory Systems

A chatbot for smallholder farmers in Kenya needs to understand "maize" differently in sentences about planting, pest control, and market pricing. Contextual embeddings make this possible without hand-crafted rules.

🌐 Multilingual Search in Ghana

Users switch between Twi, Ewe, and English in the same query. Contextual embeddings can represent code-switched text naturally, while static embeddings would treat each language as a separate space.

🎓 Low-Resource Language Models

With subword tokenization and transfer learning, contextual models can be adapted to African languages with limited data (e.g., Gemma fine-tuned on Wolof or Zulu), something impossible with static embeddings.

📈 Key Takeaway

Static embeddings are fast, interpretable, and good for simple tasks. Contextual embeddings are more powerful, handle nuance, and are essential for real-world NLP. Use static for prototyping baselines and low-resource settings; use contextual for production-quality systems.
