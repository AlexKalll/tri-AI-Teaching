BPE Tokenization in 5 Steps

Byte-Pair Encoding (BPE) is the tokenization algorithm behind GPT, BERT, Gemma, and most modern LLMs. It learns a vocabulary of subword units from data.

● TRI AI / Google DeepMind ● Session 1

The BPE Algorithm

BPE starts with individual characters and iteratively merges the most frequent adjacent pairs into new tokens.

Token
A unit of text (word, subword, or character) the model understands
Vocabulary
The set of all tokens the model knows
Merge
Combining the most frequent symbol pair into a new token
</w>
Special marker indicating end of a word boundary
OOV
Out-Of-Vocabulary — words a tokenizer cannot represent
1

Prepare Vocab

Convert corpus to word-frequency with </w> markers

Split the training text into words, count frequencies, and append </w> to each word. Break each word into its individual characters (the initial symbol vocabulary).

Example
Corpus: "low lower lowest"
Word frequencies: {"low": 1, "lower": 2, "lowest": 1}
With </w>: {"l o w </w>": 1, "l o w e r </w>": 2, "l o w e s t </w>": 1} 
def get_vocab(corpus):
    """Return dict of word->freq with  markers."""
    vocab = {}
    for word in corpus.split():
        word = ' '.join(list(word)) + ' '
        vocab[word] = vocab.get(word, 0) + 1
    return vocab
2

Count Pairs

Count adjacent symbol pairs across all words

For every word in the vocabulary, look at each adjacent pair of symbols. Count how many times each pair appears across the entire corpus.

Example
Word: "l o w </w>"
Pairs: ("l", "o"): 1, ("o", "w"): 1, ("w", "</w>"): 1

Across all words, ("l", "o") = 1 + 2 + 1 = 4 
def get_stats(vocab):
    """Count adjacent symbol pairs in vocab."""
    pairs = {}
    for word, freq in vocab.items():
        symbols = word.split()
        for i in range(len(symbols) - 1):
            pair = (symbols[i], symbols[i+1])
            pairs[pair] = pairs.get(pair, 0) + freq
    return pairs
3

Find Best Pair

Find the most frequent pair of symbols

From the pair counts, pick the pair with the highest frequency. This is the pair that will be merged into a single new token.

Example
Pair: ("l", "o") = 4 ← most frequent
Pair: ("o", "w") = 4
Pair: ("e", "r") = 2
Pair: ("w", "</w>") = 4 
def find_best_pair(pairs):
    """Return the pair with highest frequency."""
    best = max(pairs, key=pairs.get)
    return best
4

Merge

Replace all occurrences of the pair with a new symbol

Create a new token by concatenating the pair (e.g., "lo"). Replace every occurrence of the pair in every word with the new merged symbol. Update the vocabulary.

Example
New token: "lo"
Before: "l o w </w>"
After: "lo w </w>" 
def merge_vocab(pair, vocab):
    """Merge the best pair into a single token."""
    new_vocab = {}
    bigram = ' '.join(pair)
    replacement = ''.join(pair)
    for word in vocab:
        new_word = word.replace(bigram, replacement)
        new_vocab[new_word] = vocab[word]
    return new_vocab
5

Repeat

Continue until target vocabulary size is reached

Repeat Steps 2-4 iteratively. Each merge adds one new token to the vocabulary. Stop when you reach your desired vocabulary size (e.g., 32,000 for GPT-2). The final vocabulary contains individual characters plus all learned merges.

Key Insight
Initial vocab = unique characters (~256 for bytes, ~50 for chars)
Each merge = +1 token
Total merges needed = target_vocab_size - initial_vocab_size 
def bpe(corpus, num_merges):
    vocab = get_vocab(corpus)
    merges = []
    for i in range(num_merges):
        pairs = get_stats(vocab)
        best = find_best_pair(pairs)
        vocab = merge_vocab(best, vocab)
        merges.append(best)
    return vocab, merges

def merge_vocab(pair, vocab):

    bigram = ' '.join(pair)

def bpe(corpus, num_merges):
    vocab = get_vocab(corpus)
    for _ in range(num_merges):
        pairs = get_stats(vocab)
        best = find_best_pair(pairs)
        vocab = merge_vocab(best, vocab)

# Usage
corpus = "low lower lowest newer wider"
final_vocab = bpe(corpus, num_merges=10)
for token in final_vocab:
    print(token)

Vocabulary Size Tip

Larger vocabulary = fewer tokens per word but more model parameters. Smaller vocabulary = more tokens per word (slower generation) but fewer parameters. GPT-2 uses 50,257 tokens; Gemma uses 256,000. There is a trade-off between compute efficiency and model capacity.

Model Vocab Size Tokenizer Approach
GPT-2 50,257 BPE (byte-level) Byte-level BPE
BERT 30,000 WordPiece Greedy, likelihood-based
Gemma 256,000 SentencePiece Unigram LM + BPE fallback
LLaMA 32,000 SentencePiece (BPE) BPE with byte fallback
